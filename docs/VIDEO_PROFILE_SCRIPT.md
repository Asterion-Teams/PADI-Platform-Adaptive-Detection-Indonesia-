## SCENE 5 — Fitur: E-TLE Otomatis (Enforcement Engine)

**[[02:20 – 03:15]]** | **~55 detik** *(SCENE PALING PENTING)*

### Visual

- **[2:20–2:35]** Demo pelanggaran terdeteksi:
  1. Kendaraan (motor) memasuki zona busway (garis polygon merah/oranye di video)
  2. Sistem mendeteksi kendaraan dalam zona
  3. Counter dwell time mulai berjalan: "Dwell: 2s → 3s → 4s..."
  4. Threshold tercapai → **VIOLATION DETECTED** flash merah di layar
  5. Evidence capture: frame freezes, bounding box merah melingkupi kendaraan
- **[2:36–2:55]** Layar berpindah ke **Enforcement Dashboard** — violation card muncul di feed dengan animasi slide-in:
  - Thumbnail evidence (gambar pelanggaran)
  - Plat nomor: "B 1234 ABC"
  - Tipe: "Busway Occupancy"
  - Lokasi: "Simpang Dago — Camera 001"
  - Waktu: timestamp
  - Status badge: "Pending"
- **[2:56–3:05]** Klik salah satu violation → **modal detail** terbuka:
  - Foto evidence full-resolution
  - Zoom inset pada plat nomor
  - Tombol aksi: "Kirim Petugas" (hijau), "Selesai" (biru), "Tolak" (abu)
- **[3:06–3:15]** Tunjukkan **heatmap pelanggaran** di peta Leaflet — circle marker dengan intensitas warna (merah = banyak pelanggaran, kuning = sedang, hijau = sepi).

### Narasi

> *"Ketika pelanggaran terdeteksi — parkir liar lebih dari 60 detik, kendaraan masuk jalur busway, atau melawan arus — sistem langsung mencatat bukti tilang elektronik secara otomatis. Foto resolusi tinggi, lokasi GPS, timestamp kejadian, dan pembacaan plat nomor otomatis dari PaddleOCR. Operator menerima notifikasi langsung dan bisa langsung mengirim petugas ke lokasi atau memproses tilang dari dashboard."*

### Catatan Produksi

- **WAJIB punya minimal 5–10 violation record** sebelum rekam. Jalankan sistem dengan footage yang menunjukkan pelanggaran nyata atau gunakan video loop yang sudah ada kendaraan masuk zona.
- Pastikan plat nomor terbaca jelas (jika tidak ada, sementara tampilkan "Plat: B 1A234BC" sebagai contoh)
- Heatmap harus punya data — buat violation di beberapa lokasi berbeda agar dot berwarna-warni

## SCENE 7 — Fitur: ANPR & OCR Plat Nomor

**[[03:50 – 04:25]]** | **~35 detik**

### Visual

- **[3:50–4:00]** Diagram/alur visual bertahapan (animasi atau screenshot berurutan):
  1. **Vehicle Detection** — YOLO menemukan kendaraan di frame
  2. **Plate Localization** — area plat nomor di-crop (highlight merah pada crop)
  3. **Preprocessing** — crop plat di-enhance (CLAHE contrast, sharpening)
  4. **OCR Recognition** — karakter plat muncul satu per satu: "B" → "1" → "2" → "3" → "4" → "ABC"
  5. **Regional Validation** — prefix plat divalidasi berdasarkan lokasi: "B" = Jakarta
- **[4:01–4:15]** Screen recording: bukti tilang dengan plat nomor besar dan jelas, confidence score OCR (misal: 94.7%), regional prefix indicator.
- **[4:16–4:25]** Showcase pembacaan berbagai tipe plat:
  - Plat hitam (kendaraan pribadi): "B 1234 ABC"
  - Plat kuning (kendaraan umum): "B 5678 CD"
  - Confidence score terlihat di setiap pembacaan

### Narasi

> *"Sistem pembacaan plat nomor PADI menggunakan pipeline 4-tier: PaddleOCR sebagai engine utama dengan fallback ke EasyOCR dan Pytesseract. Plat yang terdeteksi langsung divalidasi berdasarkan prefix regional — B untuk Jakarta, D untuk Bandung, F untuk Bogor. Confidence score menjamin keandalan data sebelum tilang diproses."*

### Catatan Produksi

- Jika ANPR belum aktif, tampilkan diagram alur saja (lebih clean dan informative)
- Gunakan warna berbeda untuk setiap tahap pipeline di diagram
- Tunjukkan contoh plat dari berbagai wilayah jika data tersedia

## SCENE 9 — Teknologi: AI & Big Data di Baliknya

**[[05:10 – 05:50]]** | **~40 detik**

### Visual

- **[5:10–5:25]** Animasi arsitektur sistem bergulir:
  ```
  CCTV Stream → Camera Agent (daemon thread)
      ↓
  YOLOv8 Inference (GPU/CPU/ONNX fallback)
      ↓
  IoU Tracker → Enforcement Engine
      ↓
  ANPR (PaddleOCR) → Evidence JPG
      ↓
  SQLite DB + Data Lake (CSV partitioned)
      ↓
  Dashboard API → Web UI
  ```
  Setiap tahap menyala bergantian saat narasi menyebutnya.
- **[5:26–5:35]** Showcase metrik sistem:
  - "3.5M+ Data Points" (counter animasi naik)
  - "37 Concurrent Video Streams"
  - "GPU Batch Processing"
  - "99.9% Uptime"
  - "Inference < 500ms"
- **[5:36–5:50]** Model switching demo (Settings):
  - Daftar model: vehicle_v3_best.pt (custom, 52MB), yolo11m.pt (COCO), yolov5n.onnx (fallback)
  - Klik model lain — hot-swap tanpa restart
  - Tampilkan GPU info: "NVIDIA RTX ... | 8GB VRAM"

### Narasi

> *"Di balik kesederhanaan antarmuka, PADI menjalankan arsitektur Edge Big Data yang kuat. Camera Agent — daemon thread independen per kamera — menjalankan inferensi YOLO secara paralel, tracking kendaraan berbasis IoU, dan enforcement check secara simultan. Pipeline ANPR dengan 4-tier OCR fallback menjamin pembacaan plat yang andal. Data Lake partitioned per hari mendukung archival dan query historis yang efisien. Sistem otomatis recovery dari downtime dan hot-swap model tanpa interupsi."*

### Catatan Produksi

- Arsitektur flowchart: gunakan animasi sequential reveal, bukan tampilkan semua sekaligus
- Angka metrik: gunakan animasi count-up
- Tunjukkan GPU info jika benar-benar ada, atau tampilkan "CPU Mode" jika tidak ada GPU

## TIPS PRODUKSI

### Screen Recording

| Item | Rekomendasi |
|------|-------------|
| **Tools** | OBS Studio (gratis, 1920x1080) |
| **Browser** | Chrome/Edge dalam dark mode |
| **Mouse** | Gerakkan smooth, tidak terlalu cepat |
| **Timing** | Tunggu loading selesai sebelum mulai interaksi |
| **Waktu** | Rekam saat jam sibuk (07:00–09:00 atau 17:00–19:00) agar traffic terlihat nyata |
| **Audio** | Rekam audio narasi terpisah (voiceover) di ruangan tenang |

### Editing

| Item | Rekomendasi |
|------|-------------|
| **Software** | CapCut (gratis), DaVinci Resolve (gratis), Adobe Premiere |
| **Transisi** | Gunakan dissolve/fade 0.5–1 detik antar scene. Hindari transisi flash. |
| **Zoom callout** | Zoom-in 1.5–2x pada bagian penting (bounding box, plat nomor, violation alert) |
| **Text overlay** | Tambahkan label/callout teks di bagian yang perlu di-highlight |
| **Color grading** | Tone sedikit teal-orange (cinematic dark mode look) |
