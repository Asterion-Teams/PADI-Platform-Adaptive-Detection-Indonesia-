Deteksi Pelanggaran

Sistem mendeteksi 5 jenis pelanggaran secara otomatis:

Parkir Liar

Kendaraan diam >60 detik di zona larangan parkir

Jalur Busway

Kendaraan non-bus memasuki jalur busway (bus diizinkan)

Jalur Sepeda

Kendaraan bermotor memasuki jalur sepeda

Lawan Arah

Kendaraan bergerak berlawanan dengan arah yang diizinkan (>150° perbedaan)

Ngetem / Berhenti Sembarangan

Angkutan umum berhenti di luar halte

ANPR (Automatic Number Plate Recognition)

Pipeline pembacaan plat nomor:

  1. Plate Detection — YOLO model khusus mendeteksi lokasi plat di kendaraan
  2. Crop & Upscale — Area plat di-crop dan di-upscale ke 500px (LANCZOS4)
  3. Adaptive Preprocessing — Deteksi warna plat (hitam/kuning/merah/putih) → enhancement sesuai
  4. PaddleOCR — PP-OCRv4 membaca teks plat
  5. Postprocessing — Koreksi OCR error (O→0, B→8, dll) + format Indonesia

Mendukung semua jenis plat Indonesia: hitam (pribadi), kuning (umum), merah (pemerintah), putih (format baru 2022+)

Multi-Kamera & Sumber Stream

Sistem mendukung multiple kamera CCTV secara bersamaan. Setiap kamera berjalan sebagai thread terpisah dengan enforcement engine masing-masing.

  • Support HLS (.m3u8), RTSP, HTTP stream, dan file video lokal
  • Background processing — semua kamera aktif 24/7
  • Auto-reconnect dengan exponential backoff saat stream putus
  • Hot-swap model AI tanpa restart server

Kamera Dishub DKI Jakarta

Sistem terintegrasi dengan 1230+ kamera CCTV Dishub DKI Jakarta dari streetside.mugnimaestra.dev. Setiap kamera menyediakan:

  • Snapshot — Gambar JPEG di-refresh berkala oleh server Dishub
  • HLS Stream — Video real-time via .m3u8 (direkomendasikan)

Auto Token Refresh

Token autentikasi dari Dishub DKI bisa berubah sewaktu-waktu. Sistem secara otomatis mendeteksi stream yang gagal dan mengambil token terbaru dari API — tanpa intervensi operator. Maksimum 3x retry otomatis sebelum memberitahu admin.

Integrasi Hikvision

Kamera IP Hikvision (termasuk seri DeepInView/iDS untuk ANPR) dapat terhubung langsung via ISAPI protocol. Konfigurasi di Pengaturan → Hikvision:

  • ISAPI Discovery — Sistem otomatis mendeteksi device info, model, dan firmware
  • Snapshot High-Res — Ambil gambar resolusi penuh via ISAPI Streaming/channels
  • Hikvision ANPR — Kamera iDS/DeepInView mendukung pembacaan plat built-in

GPU & Performance

Optimasi performa untuk operasi 24/7:

  • GPU Batch Inference — Semua kamera share 1 GPU, inference dalam batch
  • Pre-resize — Frame di-resize ke 640px sebelum inference (hemat VRAM), koordinat di-scale balik
  • Full-res Evidence — Bukti pelanggaran tetap resolusi penuh dari kamera
  • Smart Sleep — Background kamera 2s/cycle, active view 30 FPS

Pengaturan

Semua pengaturan tersimpan permanen di data/app_settings.json dan tetap berlaku setelah restart server.

  • Confidence & IoU threshold
  • Inference image size (320-1280px)
  • Enforcement thresholds (parking, busway, cooldown)
  • X.com search query & cookies (CRM scraper)
  • Keyword pencarian berita
  • Timezone

CRM — Social Media Monitoring

Sistem memonitor mention @DishubDKI di X.com (Twitter) secara real-time menggunakan Playwright headless browser.

Cara Kerja

  1. Cookie Session — Cookies X.com di-load dari data/x_cookies.json
  2. Playwright — Headless Chromium buka X.com search dengan session yang sudah login
  3. Scrape — Extract tweet text, username, timestamp dari hasil search
  4. Filter — Hanya tweet yang relevan (mention dishub/traffic keywords) yang ditampilkan
  5. Auto-classify — Setiap mention otomatis dikategorikan (parkir liar, busway, kemacetan, dll)
  6. Import — Operator bisa import mention ke CRM sebagai report dengan 1 klik

FAQ

Apakah browser X.com harus terbuka terus?

Tidak. Cookie session berlaku berbulan-bulan (expiry 2027). Browser bisa ditutup, PC bisa shutdown. Playwright menggunakan cookie secara independen.

Kapan cookie expired?

Saat logout dari X.com, ganti password, atau X.com revoke session (~6 bulan). Jika expired, paste cookie baru di Settings.

Seberapa real-time update-nya?

Cache 2 menit. Halaman CRM auto-refresh setiap 60 detik. Tweet baru muncul dalam maksimal 2 menit.

API Reference

Semua endpoint menggunakan JSON. Base URL: http://<server>:5002

GET /api/stats

Data traffic real-time semua kamera (count, class, density, monthly aggregation).

GET /api/history?period=24h&camera_id=xxx

History traffic per interval. Periods: 30m, 1h, 6h, 12h, 24h, 7d, 30d.

GET /api/violations?status=pending&limit=50

List pelanggaran. Filter: status (pending/confirmed/dismissed), camera_id, violation_type.

GET /api/violations/executive_summary?period=24h

Executive summary: total, breakdown by type, hotspot, rekomendasi. Periods: 24h, 7d, 30d.

PATCH /api/violations/<id>

Update status violation (pending → confirmed/dismissed). Body: {"status": "confirmed"}

GET /api/zones?camera_id=xxx

List zona enforcement per kamera (polygon, type, active status).

POST /api/zones

Buat zona baru. Body: {camera_id, zone_type, geometry: [[x,y],...], name, notes}.

GET /api/crm/social_mentions

Scrape X.com untuk mention @DishubDKI. Menggunakan Playwright + session cookies. Cache 2 menit.

GET /api/crm/reports?status=open&limit=100

List CRM reports (citizen complaints). Filter by status.

POST /api/crm/reports

Submit report baru. Body: {reporter_name, description, category}. Auto-classify violation type.

GET /api/sources

List semua kamera yang terdaftar (id, name, url, lat, lng, active, external_id).

GET /api/external_cameras

Proxy server-side ke API Dishub DKI (streetside.mugnimaestra.dev). Mengembalikan 1230+ kamera dengan snapshot URL dan HLS stream URL. Cache 10 menit.

POST /api/add_camera

Tambah kamera baru. Body: {name, url, lat, lng, external_id}. Untuk kamera Dishub DKI, external_id wajib diisi agar auto token refresh berfungsi.

GET /api/settings

Semua settings saat ini (detection, enforcement, social_media, model, GPU info).

POST /api/settings

Update settings. Persist ke disk. Body: {detection: {...}, enforcement: {...}, social_media: {...}}.

POST /api/ocr/test

Upload gambar → PaddleOCR baca plat nomor. Multipart form: image file.

GET /video_feed/<camera_id>

MJPEG stream live dari kamera (dengan bounding box detection overlay).

GET /api/twitter/search?q=macet+jakarta

Cari berita lalu lintas via Google News RSS. Query configurable dari Settings.
